# Backend-приложение для автоматизации закупок

REST API сервис для автоматизации процессов закупок в розничной сети, разработанный на базе Django и Django REST Framework (DRF).

## 🚀 Описание проекта
Сервис предназначен для объединения покупателей и поставщиков в единой экосистеме. Он позволяет:
- **Покупателям:** просматривать актуальный каталог товаров, собирать корзину и оформлять заказы с доставкой.
- **Поставщикам (Магазинам):** оперативно загружать и обновлять прайс-листы в формате YAML, управлять статусом доступности магазина, отслеживать поступившие заказы.
- **Администраторам:** управлять базой пользователей, магазинами и отслеживать транзакции через Django Admin.

## 🛠 Стек технологий
- **Язык:** Python 3.10+
- **Фреймворк:** Django 5.0, Django REST Framework 3.14
- **База данных:** SQLite (базовая конфигурация)
- **Аутентификация:** DRF Token Authentication
- **Прочее:** PyYAML, requests, django-rest-passwordreset

## ⚙️ Установка и запуск (Локально)

1. **Клонируйте репозиторий:**
   ```bash
   git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
   cd orders-backend
   ```

2. **Создайте и активируйте виртуальное окружение:**
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для Linux/Mac:
   source venv/bin/activate
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Примените миграции базы данных:**
   ```bash
   python manage.py migrate
   ```

5. **Создайте суперпользователя (для доступа к ПУ):**
   ```bash
   python manage.py createsuperuser
   ```

6. **Запустите локальный сервер:**
   ```bash
   python manage.py runserver
   ```
   API будет доступно по адресу: `http://127.0.0.1:8000/api/v1/`. Админ-панель: `http://127.0.0.1:8000/admin/`.

## 📖 API Эндпоинты

### Пользователи и Аутентификация
* `POST /api/v1/user/register/` — Регистрация аккаунта
* `POST /api/v1/user/register/confirm/` — Подтверждение email по токену
* `POST /api/v1/user/login/` — Авторизация и получение Auth Token
* `GET / POST /api/v1/user/details/` — Управление профилем пользователя
* `GET / POST / PUT / DELETE /api/v1/user/contact/` — CRUD для адресов доставки
* `POST /api/v1/user/password_reset/` — Запрос на сброс пароля
* `POST /api/v1/user/password_reset/confirm/` — Подтверждение нового пароля

### Каталог
* `GET /api/v1/categories/` — Список всех категорий
* `GET /api/v1/shops/` — Список активных магазинов
* `GET /api/v1/products/` — Просмотр товаров (поддерживает фильтры `?shop_id=` и `?category_id=`)

### Корзина и Заказы
* `GET / POST / PUT / DELETE /api/v1/basket/` — Добавление/удаление/изменение количества товаров в корзине
* `POST /api/v1/order/` — Конвертация корзины в оформленный заказ
* `GET /api/v1/order/` — История завершенных заказов покупателя

### Партнерский кабинет (для type="shop")
* `POST /api/v1/partner/update/` — Загрузка/обновление прайс-листа (требуется `{"url": "ссылка_на_yaml"}`)
* `GET / POST /api/v1/partner/state/` — Управление приемом заказов магазином
* `GET /api/v1/partner/orders/` — Просмотр всех заказов, где фигурируют товары поставщика
